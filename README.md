## 前回のおさらいは、本当に一言だけ

前回の動画でObsidianの基本、つまり「ローカルに保存されるMarkdownのメモアプリで、ノート同士をリンクできる」というところまでは解説しました。まだ見ていない方は、そちらを先にご覧いただくのが一番早いです。

今回は「知ってるけど、実際どう使えばいいのかわからない」を埋める回です。前回よりも一歩踏み込んで、GPT-5.6が出たことで何が変わったのか、実際にどう活用している人がいるのかを見ていきます。

## なぜ今、この話をするのか

2026年7月9日、OpenAIはGPT-5.6を一般公開しました。Sol・Terra・Lunaという3つのモデルが用意され、コーディングやエージェント作業のトークン効率が上がったと発表されています。同じ7月9日には、これまで単独アプリだったCodexが、ChatGPT Workと合わせて新しい統合デスクトップアプリ（Chat・Work・Codexの3画面構成）に組み込まれました。Codex自体のデスクトップアプリ（ターミナル不要でフォルダを開いて使える版）はこれより少し前の2026年2月から登場していましたが、GPT-5.6という新しい頭脳と、使いやすくなった器が一つのアプリの中でつながった、というのが今回のタイミングです。

これが今回の話のポイントです。「Codexを使いこなす」のに、必ずしもターミナルの知識は要りません。アプリを開いてフォルダを選ぶだけで、Codexに自分のファイルを読み書きしてもらえます。プログラミングをしない人でも、Codexに「このフォルダの中身を整理して」「このメモを元に企画書を作って」と頼める距離にあるということです。

ただ、ここで一つ問題があります。AIに何かを頼んだ内容は、セッション（チャットの区切り）が終わると忘れられてしまいます。次に開いたときには、また一から状況を説明し直さないといけません。使えば使うほど賢くなる、というよりは、毎回ゼロから始まる感覚に近いです。

この「AIが覚えていない問題」を解決するのが、今回のテーマであるObsidianです。

## Codexに「ただのフォルダ」を渡すのと、「Obsidianのフォルダ」を渡すのは何が違うのか

ここが今回一番伝えたいポイントです。

Codexは、渡されたフォルダの中のファイルを読んだり書いたりするのが得意です。これは普通のフォルダでも、Obsidianで開いているフォルダでも変わりません。Obsidianの実体は、突き詰めれば「Markdownファイルが入ったただのフォルダ」だからです。

では何が違うのかというと、そのフォルダを人間側がどう見て、どう育てられるかです。

- **普通のフォルダ**: ファイルが増えるとどんどん探しにくくなる。どのファイルとどのファイルが関係しているのか、開いてみないとわからない。Codexが「前にこんなこと話してましたよね」と言ってくれても、それがどこから来た情報なのか人間には追いにくい。
- **Obsidianのフォルダ（Vault）**: ノート同士をリンクでつなげられる。左上のグラフビューを開くと、どのメモとどのメモが関連しているかを点と点、線でそのまま見られる。検索やタグ、プラグインで整理を手伝ってもらえる。何より、Codexが読み書きしている中身を、人間も同じ画面でそのまま確認できる。

つまり、Codexが賢くなるわけではなく、Codexに渡す「記憶の置き場所」が整理された状態になる、という違いです。実際に使っている人の話を聞くと、「AIが賢くなった」と感じる場面のほとんどは、実はこの記憶の置き場所が整ったことによるものだったりします。

## 始め方は4ステップ、そんなに難しくありません

前回のおさらいも兼ねて、今回アップデートされた点も含めて簡単に流れを書いておきます。

1. **Obsidianをダウンロードする** — 公式サイトからMac版・Windows版をダウンロードしてインストールします。
2. **Vault（保管庫）を作る** — インストール後、真っさらな状態から「Create New Vault」で新しいフォルダを一つ作ります。これがメモを貯めていく本拠地になります。
3. **ルールファイルを一つ作る** — `memory.md`のような名前で、自分の仕事内容、よく使うツール、目的、判断基準などを書いたファイルを作ります。ゼロから自分で考える必要はなく、Codexに「このObsidianのVaultを一緒にセットアップして」と頼めば、いくつか質問してくれるので、それに答えていくだけで作れます。
4. **プラグインを2つだけ入れる** — 左下の歯車からコミュニティプラグインを開き、まず「Omnisearch」（PDFや画像内の文字まで拾える検索）と「2 Hop Links Plus」（リンク先の、そのまた先（2ホップ先）までをカード形式で一覧表示する）だけ入れます。プラグインは数千種類ありますが、最初から全部入れる必要はありません。

ここまでできたら、あとは実際に使いながら育てていくだけです。

## 基本の動きは「書く・つなげる・探す・集める」の4つだけ

難しく考える必要はありません。基本操作はこの4つに集約されます。

- **書く**: 今日あったことを、左側の「デイリーノート」にとりあえず書く。完璧な文章でなくてOKです。
- **つなげる**: 関連するメモ同士を`[[ ]]`でリンクする。あとはObsidianとCodexが勝手に関連を見つけてくれることも多いです。
- **探す**: 「あの話、どこに書いたっけ」となったら検索する。Omnisearchを入れておけば、PDFの中身まで検索対象になります。
- **集める**: X（旧Twitter）で見つけた情報や、Webページの内容をそのままVaultへ入れておく。

これだけです。最初から完璧な知識ベースを作ろうとしなくて大丈夫です。

## 実際にどう使われているか、6つの実例

ここからが本題です。実際にObsidianを使っている人たちの、具体的な使い方を紹介します。1つの正解があるわけではなく、自分に合うものを見つけるための参考にしてください。

### 実例1: 自分の動画の実績を、聞くだけで教えてくれる

あるYouTubeチャンネル運営者は、動画の分析データ（再生回数やインプレッション数のCSV）をVaultの中に格納しています。ある日、Codexに「5月25日に公開した動画の結果を教えて」と話しかけたところ、Codexは自分でVaultの中を検索しに行き、「視聴回数は9,610回、インプレッションは96,234でした」と、Vault内のデータをもとに正確な数字で答えてくれました。

前提情報を何も与えていないのに、Vaultの中を見に行って答えてくれる。これがObsidianと連携している状態です。過去のデータをいちいちCodexに貼り付ける必要がなくなります。

### 実例2: 続きものの相談を、毎回ゼロから説明しなくていい

展示会の準備のように、一回の相談では終わらず、何度も壁打ちを繰り返す作業があります。普通なら「うちの会社が出展する展示会は、目的がこうで、こういう内容で……」と、相談するたびに前提を説明し直すことになります。

Vaultに一度この情報を記憶させておくと、次に相談するときには「展示会について」と言うだけで、Codexが「目的はこうで、スケジュールはこうでしたね。今回はどの点を詰めたいですか？」と、前回までの内容を踏まえた状態で会話が始まります。壁打ちを重ねるたびに、その内容もVaultへ蓄積されていきます。

### 実例3: 制作物の変更の経緯を、全部覚えている

Webページ（LP）の制作を例にすると、最初のたたき台を作り、社内のメンバーからフィードバックをもらい、A/Bテストの結果を見て修正する、というやり取りを何度も繰り返すことがあります。

この一連のやり取りをCodexとの会話で進めていくと、その内容は自動的にVault内の該当ファイルへ保存されていきます。「前はどういう経緯で今の内容になったんだっけ」と聞けば、過去の判断まで遡って確認できます。人間が議事録を別に残しておく必要がありません。

### 実例4: タスク管理を全部、AIとの会話に任せる

あるライターは、Obsidianを直接開くことはほとんどありません。その代わり、Codexに「このタスク終わりました」「タスク一覧を見せて」「この締切が変わりました」と話しかけるだけで、Codexがその内容をVault内のタスク管理用ファイルに反映してくれます。

一般的なタスク管理ツールだと、追加ボタンを押して、タイトルを入れて、プロジェクトをプルダウンで選んで……という入力の手間がありますが、これがすべて会話だけで済みます。「動画系のタスクだけ抽出して」と頼めば、それも一覧にして出してくれます。頭の中にあることをそのまま話すだけで整理される感覚です。

### 実例5: AIにインタビューされる日記で、三日坊主を卒業する

日記や振り返りが続かない人向けの使い方もあります。あるユーザーは、自分でメモを書く代わりに、Codexに「日記を書こう」と話しかけると、「今日は何をしましたか？」「うまくいったこと、詰まったことはありますか？」と質問してくれるモードを作っています。答えるだけで、その日の記録がVaultへ整った形で残っていきます。

さらに、週末になると「今週はこんな一週間でしたね」と、その週の記録を読み返して振り返りを促してくれる仕組みも組んでいます。自分で書く負担がないので、続けやすいのが特徴です。

### 実例6: AIを使わない使い方も、もちろんアリ

ここまでCodexとの連携を中心に紹介してきましたが、Obsidianの機能だけで運用している人ももちろんいます。あるユーザーは、フォルダを深く分けるのをやめて、メモに「タグ」を付けるだけの運用にしています。ホーム画面に自分でボタンを作っておき、それを押すと該当するタグのメモだけが一覧で表示される、という仕組みです。フォルダを何度も開いて目的のメモを探す必要がなくなります。

このユーザーは、以前AIと連携させる運用も試したものの、「AIのために記憶を整理していたはずが、結局は人間が読みやすい形に最適化されてしまい、どちらのためにもならなかった」という理由でやめたと話しています。全員がAIと連携させる必要はなく、Obsidian単体でも十分に便利だという例です。

## 情報収集を、溜めっぱなしにしない

Xやブックマークで「あとで読む」を溜め込んで、結局読み返さない、という経験がある人は多いと思います。Obsidianを使うと、この「集めたけど使わない」状態を減らせます。

- **英語の発表を日本語で要約して格納する**: 海外AIサービスの発表は英語であることが多く、動画だと特に何を言っているのかわかりにくいものです。要約AIサービスに動画を読み込ませて日本語の図解を作らせ、それをそのままVaultの「Xリサーチ」用フォルダへ入れておくと、必要なときにCodexがそこから引っ張ってきてくれます。
- **Web Clipperでワンクリック保存する**: Obsidian公式のブラウザ拡張機能を入れておくと、気になったWebページをボタン一つでMarkdown形式に変換してVaultへ保存できます。コピペしてMarkdownに整形する手間がありません。

## 育てていくと、何が変わるか

使い始めてすぐに劇的な変化があるわけではありません。実際に使っている人の声を集めると、だいたい次のような時間感覚のようです。

- **1週間目**: 貯めた知識をもとに、基本的な受け答えができるようになる段階。
- **1ヶ月目**: 取引先の情報、チーム構成、業務の流れなどをひととおり把握している状態になる。
- **2〜3ヶ月目**: 人間側が忘れていたことを、AIの方から「これも関係ありますよ」と指摘してくれるようになる。

もう一つ重要なのが、間違いを記録する習慣です。AIの出力がイメージと違ったときに、「次からはこうしてください」という一言を残しておくと、同じ間違いを繰り返さなくなります。この「反省メモ」を溜めていくことが、精度が上がっていく一番の近道です。逆に、古くなった情報やもう使わなくなったルールは、定期的に見直して削っていく必要があります。足すだけでは、いずれ矛盾した情報だらけになってしまいます。

## セッションが切れても、複数のAIをまたいで使える

Codexには1回のセッションで使えるトークン数の上限があります。長く使っていると、だんだん出力の精度が落ちていく感覚を持ったことがある人もいると思います。

ここでObsidianが役立ちます。記憶がVaultという外部の場所にあるので、Codexのセッションが切れても、Claude CodeやCursor、Antigravityなど別のAIエージェントに切り替えれば、同じVaultを読みに行って会話を再開できます。どのAIを使うか迷う前に、まずこのVaultをしっかり育てておくことの方が重要です。

使っている人たちの間では、それぞれの得意分野で使い分ける声も多く聞かれます。Claude Codeは日本語の質と出力の丁寧さ、Codexはスピードとレビューの速さ、Cursorはコストパフォーマンス、Antigravityは他のGoogleサービスとの連携、といった評判です。どれか一つに絞る必要はなく、Vaultという共通の記憶を軸にして、場面ごとに使い分けるという考え方です。

## こんな一言で、こう変わる

具体的にどんな話しかけ方をすればいいのか、短い例をいくつか挙げておきます。

- 「今週の会議の決定事項と宿題をまとめて」→ Vault内の会議メモを横断して拾い上げ、担当と期限つきで一覧にしてくれる。
- 「先月のクライアントAとのやり取り、覚えてる？」→ 該当するプロジェクトのファイルを探しに行き、経緯を要約して答えてくれる。
- 「このアイデア、前にも似たようなこと考えてなかったっけ」→ 関連するメモをリンクをたどって探し出し、「これと似ていますね」と提示してくれる。
- 「今日はここまで。続きは来週」→ 現在の状況を一行でVaultへ書き残してくれるので、次回開いたときに「前回はここまででしたね」から会話が始まる。

いずれも、事前に長い説明をしていません。Vaultに情報が蓄積されているからこそ、短い一言で意図が伝わるようになります。

## 気をつけたいこと

便利な分、気をつけておきたい点もあります。

- **ファクトチェックは必須**: Web上の情報やAIの調べ物には間違いも混ざります。間違った情報がVaultに格納されると、その後の回答もずっと間違ったままになりかねません。ただ、これも毎回自分でやるのは大変なので、最初のルールファイルに「格納前に確認する」という指示を組み込んでおくと、負担を減らせます。
- **完璧を目指さない**: 最初から完璧なファイル構造を作ろうとすると、続きません。まずは今日あったことを書くところから始めて大丈夫です。散らかったメモはAIが後から整理を手伝ってくれます。
- **秘密情報はVaultに書かない**: パスワードやAPIキーなど、外部に漏れると困る情報はVaultへ保存しないようにしましょう。

## プラグインは、最初の2つだけで十分

Obsidianにはコミュニティプラグインが数千種類ありますが、最初から手を出しすぎないことをおすすめします。まずは前述の「Omnisearch」と「2 Hop Links Plus」の2つだけを入れて、しばらく使ってみてください。「これが足りない」と感じてから、必要なプラグインを一つずつ足していく方が、失敗が少なくなります。プラグインを一度にたくさん入れると、何か不具合が起きたときに原因を特定しづらくなる、という声もよく聞かれます。

## 今日からできる、最初の一歩

1. Obsidianをダウンロードして、Vaultを一つ作る。
2. Codexに「このVaultをセットアップして」と頼み、ルールファイルを作ってもらう。
3. Omnisearchと2 Hop Links Plusの2つだけプラグインを入れる。
4. 今日あった出来事を、デイリーノートに書いてみる。
5. 次にCodexへ何かを頼むとき、このVaultのフォルダを指定して会話を始めてみる。

完璧な形を最初から目指す必要はありません。まずは1つの作業を、Vaultを介してCodexと一往復してみることから始めてみてください。
